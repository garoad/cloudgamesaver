# 📝 CloudGameSaver 기능 명세서 (Functional Specification)

## 1. 개요
CloudGameSaver는 여러 대의 PC에서 게임 세이브 데이터의 위치가 다르더라도, **Dropbox 클라우드**를 매개체로 하여 항상 최신 상태의 세이브 파일을 유지할 수 있도록 돕는 양방향 자동 동기화 도구입니다.

## 2. 핵심 기능 (Core Features)

### 2.1 드롭박스(Dropbox) 연동 및 인증
*   **OAuth2 인증**: `PKCE` 기반의 OAuth2 흐름을 사용하며, 로컬 서버(`localhost:8421`)를 통해 인증 코드를 수신합니다.
*   **영구 세션 유지**: `Refresh Token`을 도입하여 액세스 토큰 만료 시 자동으로 갱신합니다. (최초 1회 로그인 후 반영구적 사용 가능)
*   **토큰 보안**: 갱신된 토큰은 로컬 스토리지 및 앱 상태에 실시간으로 반영됩니다.

### 2.2 지능형 양방향 동기화 (Smart Sync)
*   **무결성 검증**: Dropbox 표준 해시 알고리즘(4MB 블록 단위 SHA256)을 사용하여 파일의 실제 내용 일치 여부를 판별합니다.
*   **최신성 판단**: 로컬 파일의 수정 시간과 Dropbox 서버의 `server_modified` 시간을 비교하여 더 최근에 수정된 파일을 선택합니다.
*   **병렬 처리**: `FuturesUnordered`를 사용하여 최대 5개의 파일을 동시에 업로드/다운로드하여 속도를 최적화합니다.
*   **자동 폴더 생성**: 클라우드나 로컬에 경로가 없을 경우 동기화 과정에서 자동으로 생성하여 에러를 방지합니다.
*   **작업 취소**: 동기화 진행 중 사용자가 언제든지 안전하게 작업을 중단할 수 있는 취소 기능을 제공합니다.

### 2.3 게임 관리 (Game Management)
*   **자동 감지**: 드롭박스 루트 디렉토리의 폴더들을 스캔하여 새로운 게임 세이브 폴더를 자동으로 목록에 추가합니다.
*   **수동 등록**: 사용자가 직접 게임 이름과 로컬 세이브 폴더 경로를 지정하여 등록할 수 있습니다.
*   **활성화 제어**: 각 게임별로 동기화 활성화/비활성화 토글을 제공하여 필요한 게임만 선택적으로 동기화할 수 있습니다.
*   **경로 설정**: 로컬 폴더 선택 다이얼로그를 통해 직관적으로 세이브 경로를 연결할 수 있습니다.

### 2.4 자동 업데이트 시스템 (Auto-Updater)
*   **버전 체크**: 앱 시작 시 및 실행 중 30분 간격으로 GitHub Release에 배포된 최신 버전을 확인합니다.
*   **원클릭 업데이트**: 새 버전 발견 시 변경 사항을 표시하고, 사용자 승인 시 자동으로 다운로드 및 재시작하여 설치를 완료합니다.
*   **진행률 표시**: 업데이트 파일 다운로드 진행 상황을 실시간 퍼센트 단위로 피드백합니다.

## 3. 사용자 인터페이스 (UI/UX) - 유지 대상

### 3.1 메인 화면 구성
*   **Header**: 앱 타이틀 및 서비스 설명.
*   **Settings Section**: 드롭박스 연결 상태 확인, 목록 갱신 버튼, 수동 업데이트 확인 버튼.
*   **Add Section**: 새로운 게임을 등록하기 위한 입력 폼 (이름 입력 및 폴더 선택).
*   **List Section**: 등록된 게임 카드 목록. (활성화 토글, 경로 정보, 연결/삭제 버튼 포함)
*   **Action Section**: 전체 동기화 시작 버튼 및 실시간 상태 로그 표시창.

### 3.2 로딩 오버레이 (Loading Overlay)
*   동기화 또는 업데이트 진행 시 화면 전체를 덮는 오버레이 표시.
*   실시간 진행바(ProgressBar), 현재 처리 중인 파일명, 완료 퍼센티지 표시.
*   동기화 중 작업을 중단할 수 있는 '취소' 버튼 배치.

## 4. 기술 스택 (Technical Stack)
*   **Framework**: Tauri v2 (Rust-based Desktop Framework)
*   **Frontend**: React 19, TypeScript, Vite, Vanilla CSS
*   **Backend**: Rust (Tokio, Reqwest, Serde, Sha2)
*   **Storage**: LocalStorage (설정 및 토큰 저장)
*   **CI/CD**: GitHub Actions (Multi-platform Build & Release)
